Elementos HTML

ESTRUCTURA BÁSICA
Elemento Descripción Documentación
html Representa la raiz del documento HTML. <html>
head Provee información general (metadatos) acerca del documento, incluyendo su título y enlaces a scripts y hojas de estilos. <head>
body Representa el contenido de un documento HTML. Solo puede existir uno en el documento. <body>
ELEMENTOS GENÉRICOS
Elemento Descripción Documentación
div Sirve para crear secciones o agrupar contenidos. Contenedor en bloque <div>
span Contenedor en línea. Sirve para aplicar estilo al texto o agrupar elementos en línea. <span>
ELEMENTOS SEMÁNTICOS
Elemento Descripción Documentación
header Grupo de ayudas introductorias o de navegación. Puede contener algunos elementos de encabezado. <header>
article Representa una composición auto-contenida en un documento, página, una aplicación o en el sitio. <article>
section Sirve para determinar qué contenido corresponde a qué parte de un esquema. <section>
nav Representa una sección de una página cuyo propósito es proporcionar enlaces de navegación. <nav>
aside Representa una sección de una página; contenido que está indirectamente relacionado con el contenido principal. <aside>
footer Representa un pie de página para el contenido de sección más cercano o el elemento raíz de sección. <footer>
main representa el contenido principal del <body> de un documento o aplicación. <main>
MANEJO DE TEXTO
Elemento Descripción Documentación
h1 - h6 Un elemento de encabezado describe brevemente el tema de la sección que presenta. <h1> - <h6>
p Es el apropiado para distribuir el texto en párrafos (crea caja en bloque). <p>
br produce un salto de línea en el texto (retorno de carro). <br>
hr Representa un cambio de tema entre párrafos (normalmente representado con una línea). <hr>
wbr Representa una posición dentro del texto donde el explorador puede opcionalmente saltar una línea. <wbr>
sup Define un fragmento de texto que se debe mostrar, más alto, y más pequeño (superíndice). <sup>
sub Define un fragmento de texto que se debe mostrar, más bajo, y más pequeño (subíndice). <sub>
strong Es el apropiado para marcar con especial énfasis las partes más importantes de un texto. <strong>
em Es el apropiado para marcar con énfasis las partes importantes de un texto. <em>
mark Representa un texto marcado o resaltado como referencia o anotación. <mark>
q Indica que el texto adjunto es una cita corta en línea. <q>
blockquote Crea citas en bloque, marca las citas a otros autores o documentos. <blockquote>
cite Marca una referencia a una fuente, o el autor de un texto citado. <cite>
dfn Sirve para marcar el término que se quiere definir. <dfn>
abbr Representa una abreviación o acrónimo. <abbr>
small hace el tamaño del texto una talla más pequeña que el tamaño mínimo de fuente del navegador. <small>
ins Marca las partes de un texto que han sido añadidos al documento. <ins>
s representa contenido que ya no es relevante o preciso (tachado). <s>
code Es el apropiado para marcar el código de un programa. <code>
samp se utiliza para incluir texto en línea, mostrando un programa de ordenador. <samp>
kbd Marca el texto que debe introducir el usuario. <kbd>
pre Representa texto preformateado. <pre>
address Aporta información de contacto para su <article> más cercano o ancestro <address>
Caracteres especiales Permiten imprimir caracteres (< >) sin afectar al código. Documentación
